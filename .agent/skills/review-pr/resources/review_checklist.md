# PR Review Checklist

## 코드 품질
- [ ] 네이밍이 명확하고 중복 코드가 없는가?
- [ ] 함수 길이가 적절한가? (30줄 이하 권장)

## 로직 & 에러 처리
- [ ] 엣지 케이스(빈 값, null, 경계값) 처리가 되어 있는가?
- [ ] 에러 핸들링이 적절한가?

## 보안
- [ ] 하드코딩된 시크릿이 없는가?
- [ ] 사용자 입력 검증 및 인젝션 방어가 되어 있는가?

## 성능
- [ ] 불필요한 반복문, N+1 쿼리 문제가 없는가?
- [ ] 대용량 처리 시 메모리/캐싱이 고려되었는가?

## 테스트 & 문서
- [ ] 새 기능에 대한 테스트가 추가되었는가?
- [ ] 공개 API의 docstring과 관련 문서가 업데이트되었는가?
